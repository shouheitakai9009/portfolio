# ディレクトリ構成

- docs
  - `directory-structure.md`: 全般的なディレクトリ構成
- src
  - `assets`: 画像やフォントなどの静的アセット
  - `data`: JSON やその他のデータファイル
  - `designs`: 汎用的に使用できるドメイン知識を持たない純粋な UI コンポーネント
  - `widgets`: 特定のドメイン知識を持つ UI コンポーネント
  - `states`: アプリケーションの状態管理に関連するコード
    - index.ts: ストアの設定/ユーティリティ関数の提供
    - slice.ts: ビジネスロジックの実装
    - state.ts: 初期状態とグローバル状態の型定義

## コロケーション

UI コンポーネントの配置は以下のようにコンポーネントとスタイル、処理を同階層に配置してください。

```
src/
  designs/
    Button/
      index.tsx
      index.css.ts
  widgets/
    Header/
      index.tsx
      index.css.ts
      useDrawerMenu.ts
      MenuButton/
        index.tsx
        index.css.ts
```

## 凝集

各ディレクトリは高い凝集度を持つように設計されています。例えば、`designs` ディレクトリにはドメイン知識を持たない純粋な UI コンポーネントが含まれ、`widgets` ディレクトリには特定のドメイン知識を持つ UI コンポーネントが含まれます。これにより、コードの再利用性と保守性が向上します。

各ファイルは凝集度を高く保ちます。例えば、コンポーネントのロジック、スタイル、フックは同じディレクトリに配置され、関連するコードが一箇所にまとまるようにします。
さらにコンポーネントが大きくなる場合は、サブコンポーネントを同じディレクトリ内に配置します。
